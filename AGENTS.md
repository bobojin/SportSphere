# rule

如果用户输入的指令不够明确，要根据用户的输入判断用户真实意图，根据对意图的判断来考虑需要完成的是什么。

如果无法明确判断，或者涉及到关键的选择，向用户进行确认。

## 构建产品的标准

- 前后端分离，走构建流程
- 数据库用sqlite
- 图标用material design风格的
- 需要初始化的示例数据
- 页面样式 按照 DESIGN.md 进行
- 页面上的功能应该是自解释的，不能过于依赖文案性的使用说明
- 要进行合理的模块拆分，页面拆分，必要的地方可以使用弹窗，避免在一个页面放太多的东西
- 要避免页面出现 纵向 拉伸的情况
- 要有加载中状态
- 下拉选择 或 下拉多选，不要使用原生的，下拉选择的箭头的右边 需要有间距

## 其他

- 自动生成 启动、停止、重启 三个bat文件，停止 或 重启 交给用户手动操作就可以。
- 不要走备用端口验证
